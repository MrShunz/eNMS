Multiprocessing allows work for multiple devices to run concurrently.

Use the checkbox to enable/disable multiprocessing for a service or workflow. The Maximum Number of Processes field controls the limit of how many devices can run concurrently. Higher concurrency makes sense for operations that spend a lot of time waiting for data from devices, lower concurrency for compute focused activities. Actual performance varies based on other activities running on the same system.

Contexts where multiprocessing adds value
  • Services in a service by service workflow or subworkflow
  • Device by device workflows and sub-workflows
Contexts where multiprocessing does not add value
  • Service by service workflows
  • Services or sub-workflows within a device by device workflow
